Writeup for Sprite Viewer

Category: Web
Author: Tom

You can view a video version of this writeup here

Sprite Viewer was a challenge that made up part of the of the 2020 Riceteacatpanda CTF and was also my favourite challenge out of all the ones I did during the CTF. The challenge description talked about the flag format and provided a link to a Unity WebGL game that showed us a few different animated sprites. We needed to get the sprite image of the goblin, and the MD5 hash of that file would be the flag.

If we look at the Unity WebGL manual page for building and running a game, there are two sections of interest:

  • The Build folder contains the following files [...] A MyProject.data.unityweb file containing the Asset data and Scenes
  • The contents of the *.unityweb files in the Build folder may be compressed with gzip, brotli or may be uncompressed, depending on the Publishing Settings.

So, we need to look for a .data.unityweb file. Reloading the page while looking at the network tab of Firefox's devtools reveals this file, which we can download and decompress using 7-Zip.

With this, we can open UnityAssetStudio and use that to extract the goblin sprite image, which we can then MD5 hash using a Python script that I wrote.

I got the 21st solve on this one too, even on the fourth day of the CTF, which I'm super proud of!
