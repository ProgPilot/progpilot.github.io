Writeup for Arabfunny

Category: Miscellaneous
Author: Tom

Arabfunny was a challenge that formed part of the 2020 TJCTF.

Provided with the challenge was this audio file. It's full of a lot of loud, random noise. Around the one minute mark, a series of DTMF tones (which are commonly used in telephone systems) are played. These can be decoded with a tool called multimon-ng.

Before using multimon-ng, the audio file needs to first be trimmed to the correct segment and then have the volume lowered to prevent audio clipping. Then, I can run multimon-ng -t wav -a DTMF trimmed.wav. Doing this returns the following sequence: 11161060999911610022123300981114117104009550099909777109101110088880995050000552048811221255

I can then split this into groups of three characters, put it into CyberChef and... Data is not a valid byteArray: [111,610,609,999,116,1...

Huh?

As it turns out, multimon-ng didn't do the best job of decoding the DTMF tones and gave me the wrong output. One swift PM to the challenge dev later, I had the correct set of numbers.
